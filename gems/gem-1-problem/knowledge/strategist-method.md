# Strategist — Method

The Strategist evaluates whether a proposed direction strengthens the product's core purpose. Applies four tests.

---

## When to apply the Strategist lens

Apply when:
- Scope fit is unclear (does this belong in the product at this stage?)
- A direction would expand the product's surface area significantly
- The problem feels real but may serve a different product better
- Pressure to add a feature comes from competitive comparison, not user behavior

---

## The four tests

### 1. Product purpose test

Name the product's core purpose in one sentence. Not what it does — what it is.

Examples:
- "A recording instrument for thought"
- "A clarity tool for people who think by writing"
- "A structured capture system for active thinkers"

Ask: does solving this problem deepen that core purpose, or expand away from it?

Deepen: the product becomes more itself. Users get more of what they came for.
Expand: the product becomes something adjacent. The new direction requires a different mental model.

Expansion is not always wrong. But it must be a deliberate choice, not an accretion.

### 2. Interaction cost test

Every feature has an interaction cost: the cognitive load it adds to the product even when unused. A settings toggle costs something. A new navigation destination costs something. A new concept the user must learn costs something.

Ask: is the value of solving this problem greater than the interaction cost it adds to the product?

If the answer is not clearly yes: the problem may belong in a different product, or may not be ready for this one.

### 3. Precedent test

Has this class of problem been solved before in analogous products? What happened?

The precedent test prevents repeating known mistakes. It does not prevent trying things that failed elsewhere — different contexts produce different outcomes. But it requires that prior failure be acknowledged and the reason for different outcome here be stated.

### 4. Drift test

Look at the last three features shipped. Is there a pattern of expansion? Is the product drifting from its core purpose?

A single expansion is a direction. Three expansions is drift. The drift test catches gradual, invisible scope creep that no single decision reveals.

---

## Strategist outputs

The Strategist does not produce a verdict alone — it feeds into the PM's four-move workflow.

What it produces:
- A pass/fail on each test (not a score — a judgment)
- The specific tension when a test fails or is borderline
- A recommendation when fit is unclear: does this belong here, or somewhere else, or not yet?

What it does not produce:
- A recommendation on implementation approach
- An evaluation of feasibility
- Design directions
